# Go Bug Fix Skill

本 Skill 用于 Go 项目的 Bug 修复、异常排查、测试失败修复、CI 失败修复和线上问题定位。

公共开发底线遵守项目根目录中的：

```text
AGENTS.md
CLAUDE.md
```

本 Skill 只补充“Bug 修复”场景下的额外规则。

---

## 1. 适用场景

当用户要求以下任务时，使用本 Skill：

```text
修复 Bug
排查 panic
修复测试失败
修复 CI 失败
修复接口返回异常
修复并发问题
修复数据不一致
修复配置不生效
修复超时、重试、连接泄漏问题
修复性能退化
```

---

## 2. Bug 修复原则

修 Bug 必须遵守：

1. 先复现或确认现象，再修改。
2. 先定位根因，不只修表象。
3. 优先最小修复，不扩大改动范围。
4. 不借修 Bug 之名做大规模重构。
5. 不为了测试通过而删除测试。
6. 不为了编译通过而注释核心逻辑。
7. 修复后必须补充或更新回归测试。
8. 如果无法复现，必须说明当前证据和推断边界。

---

## 3. 修复前必须确认

修复前必须先输出：

```text
问题现象：
xxx

已知信息：
- 报错：xxx
- 日志：xxx
- 失败测试：xxx
- 相关接口：xxx

相关代码位置：
- xxx
- xxx

初步判断：
xxx

可能根因：
1. xxx
2. xxx

本次先验证：
xxx

预计修改文件：
- xxx
- xxx

验证方式：
- 复现命令：xxx
- 回归测试：xxx

请确认是否开始。
```

如果信息不足，不能直接猜着改，必须先说明：

```text
当前无法确认根因，建议先查看：
- xxx
- xxx
```

---

## 4. 定位顺序

排查 Bug 时优先按以下顺序：

```text
1. 读取报错、日志、失败测试
2. 找到直接失败位置
3. 向上追踪调用链
4. 确认输入、状态、配置、依赖是否符合预期
5. 找到最小复现路径
6. 判断根因
7. 制定最小修复方案
8. 补充回归测试
9. 执行验证
```

不要在未定位调用链前直接大改。

---

## 5. 修复范围约束

Bug 修复只允许修改与根因直接相关的代码。

禁止：

```text
顺手重构整个模块
顺手改目录结构
顺手升级依赖
顺手改 API
顺手改配置格式
顺手删除旧逻辑
顺手重写测试体系
```

允许：

```text
补充必要错误处理
补充必要边界判断
补充必要日志上下文
提取极小范围私有函数
新增回归测试
```

---

## 6. 回归测试要求

修复 Bug 后必须尽量补充测试。

优先补：

```text
失败场景测试
边界条件测试
错误分支测试
并发场景测试
配置缺失测试
外部依赖失败测试
```

如果无法补测试，必须说明原因：

```text
未补充回归测试，原因：
xxx

建议后续补充方式：
xxx
```

不得只说“已修复”而没有验证。

---

## 7. 常见 Bug 检查清单

排查时重点检查：

```text
nil pointer
数组 / slice 越界
map 并发读写
goroutine 泄漏
channel 死锁
context 未传递
timeout 缺失
错误被吞掉
配置默认值不正确
环境变量未读取
请求参数未校验
响应结构不兼容
Redis Key 不一致
PromQL 拼接错误
HTTP client 未设置超时
资源未关闭
测试依赖真实外部服务
```

---

## 8. 日志和错误处理

修复 Bug 时可以补充必要日志，但必须遵守：

```text
日志必须有上下文
不能打印敏感信息
不能留下 fmt.Println 调试日志
不能用 panic 代替错误处理
不能吞掉错误
```

推荐错误包装：

```go
if err != nil {
    return fmt.Errorf("query prometheus: %w", err)
}
```

---

## 9. 修复后输出格式

每完成一个 Bug 修复模块后，输出：

```text
本次修复：
xxx

根因分析：
xxx

修复方式：
1. xxx
2. xxx

修改文件：
- xxx
- xxx

回归测试：
- 新增 / 修改：xxx
- 覆盖场景：xxx

已执行验证：
- 复现命令：通过/失败
- goimports：通过/失败
- go test：通过/失败
- go vet：通过/失败

未执行：
- xxx，原因：xxx

风险与注意：
- xxx

建议提交：
git add xxx
git commit -m "fix: xxx"

请确认是否继续。
```

---

## 10. 禁止行为

Bug 修复中禁止：

```text
1. 未复现就大改
2. 未定位根因就猜测修改
3. 只改测试不改真实问题
4. 删除失败测试来制造通过
5. 注释核心逻辑来制造编译通过
6. 把小 Bug 扩大成重构任务
7. 引入无关依赖
8. 改变 API 行为却不说明
9. 不执行回归验证
10. 假称问题已修复
```
